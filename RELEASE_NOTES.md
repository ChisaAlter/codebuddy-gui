# CodeBuddy GUI 0.1.0

首个可安装预览版本，面向本地多项目 CodeBuddy CLI 工作流。

## 已提供

- 多项目和多对话持久化，支持并行 CodeBuddy 运行时与后台会话。
- 文件浏览、Monaco 编辑器、项目终端和 Git 工作流。
- 插件、Worker、任务、实例、日志、指标、链路和远程渠道管理。
- Windows 托盘、单实例唤回、窗口状态恢复和可选安装目录。
- 次级页面按需加载，编辑器仅在进入工作区后加载。
- 首次启动时若 CodeBuddy CLI 不可用或项目目录已移动，会显示可操作的明确错误信息。
- 应用界面、Windows 产物和 ACP 客户端上报统一使用同一份产品名称与版本。
- 使用统计适配当前 CodeBuddy 数据契约，可查看会话、消息、活跃天数、模型 Token、常用工具和近期活动。
- 远程渠道仅显示当前 CLI 支持的操作；微信解绑后可直接重新扫码绑定。
- 系统监控展示当前 CLI 的实例健康、PID、目录、版本、运行时长和内存占用。
- 插件页兼容单个插件响应，并展示安装范围、作者、许可、技能数量和市场信息。
- Worker 页展示当前实例、Endpoint、模式和心跳，并避免误终止当前项目运行时。
- 快捷键管理支持搜索、上下文筛选、用户覆盖、后端校验、恢复单项默认和全部重置。
- 设置页补齐 Token 计数、通知、信任、状态栏和可信目录，并正确保存与回滚嵌套设置。
- 项目重命名和移除改为应用内确认流程，明确提示未保存修改，并在状态持久化失败时恢复操作前状态。

## 使用要求

- Windows x64。
- 本机可直接运行 `codebuddy` CLI，并已完成其必要配置。

## 数据保留

- 覆盖安装会保留现有项目和对话数据。卸载程序默认保留用户数据，方便后续重装恢复。
- 产品状态保留最近一次有效备份；主状态文件损坏时自动恢复备份，并保留原始坏文件用于诊断。

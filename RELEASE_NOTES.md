# CodeBuddy GUI 1.0.1

1.0.1 是面向 CodeBuddy CLI 2.122.0 的桌面稳定性修复版本，重点收口新会话发送、长任务、权限卡、偏好持久化和侧栏快捷键。

## 开发主干（未发版）

- 设置页 CLI 维护增加兼容状态（最低 / 推荐版本）；低于最低版本或 CLI 缺失时 **硬阻断** 项目运行时启动，并提供一键安装推荐版本。
- 关键路径补充集成测试：新会话 sessionId、权限卡去重响应、退出同步 flush、product-state 磁盘恢复。
- 渐进拆分 `store.js`：抽出 helpers 与 slices（product-persist、projects-runtime、sessions-chat），对外 `useStore` API 不变。

## 本版本重点

- 修复新会话刚连接时线程记录尚未写入 sessionId，首条消息可能以 `null` 调用 `session/prompt` 并返回 `Invalid params` 的竞态。
- 长时间运行的 `session/prompt` 不再受 120 秒空闲超时限制；普通 ACP 请求仍保留 30 秒超时边界。
- 权限请求同时按 `interruptionId` 和 `toolCallId` 去重，旧会话的迟到事件不会再写入当前线程。
- GUI 主题、提示建议、桌面通知等偏好写入 `product-state.json`，切换项目或重启后会恢复，不再依赖动态本地端口对应的 localStorage。
- `Ctrl/Alt/Meta` 组合快捷键在聊天输入框、Monaco 和终端聚焦时仍可执行；无修饰键和仅 Shift 的绑定继续让位于正常文本输入。
- 侧栏折叠宽度固定为真实 60px，展开宽度保持 220px 到 252px；移除会被后台窗口节流的布局过渡，快捷键切换立即生效。
- 桌面验收夹具同步当前 19 个路由和运行时就绪日志，使用隔离产品状态与确定性控件触发，避免后台窗口、高 DPI 和旧选择器造成误报。

## 1.0.0

1.0.0 首个稳定版本，完成编辑器资源精简、静态质量收口和 Windows 本地发布链路整理。

- Monaco 改为核心编辑器、单一 editor worker 和实际使用语言的按需高亮，不再打包 TypeScript、CSS、HTML 和 JSON 语言服务 worker。
- 前端 assets 从约 14.85 MB 降至约 5.18 MB，文件数量从 113 个降至 32 个；工作区仍支持 JavaScript、TypeScript、JSON、Markdown、Python、CSS、HTML、YAML、Shell 和纯文本。
- JSON 使用轻量 tokenizer 保留注释、括号、引号自动闭合和语法着色，不引入完整 JSON language service。
- ESLint 正确识别 JSX 变量和项目既有 `_` 占位参数，清理真实死代码与异常空白，完整 lint 达到 0 error / 0 warning。
- GitHub Actions 继续保持关闭，Windows 安装包和更新清单由本地发布命令生成，不依赖 CI。

## 0.9.0

0.9.0 补齐 CodeBuddy CLI 指定版本安装与回滚。

- 设置页 CLI 维护区新增“安装版本”，支持 `latest`、完整版本号和带 `v` 前缀的版本输入；填写旧版本号即可执行回滚。
- 安装前使用独立应用内对话框确认目标，渲染层与主进程分别校验版本格式，不允许把任意文本拼入 CLI 命令。
- 安装调用真实 `codebuddy install <target>`，最长运行 10 分钟，并与诊断、更新和插件维护共用独占操作锁。
- 安装完成后重新读取 CLI 版本，显示安装前后版本和真实命令输出；输出过长时保留最新 1 MB。
- 无论版本是否变化，安装完成后都会提示重启当前项目运行时，避免继续使用安装前已启动的进程。

## 0.8.0

- 后台会话列表新增“接管”按钮，点击前重新读取 `codebuddy ps --json`，确认 PID 仍存在且类型为后台任务。
- 接管通过真实 `codebuddy attach <pid>` 完成，优先在 Windows Terminal 新标签页中打开；Windows Terminal 不可用时回退到独立 Windows PowerShell 窗口。
- 交互终端使用后台会话自己的工作目录；PID 和目录均在主进程重新校验，不接受渲染层直接拼接命令。
- 接管与启动、终止操作共用同步操作锁，避免快速重复点击；GUI 明确反馈终端类型、打开失败和“关闭窗口不会终止后台任务”。
- 交互会话、Daemon 和预热进程继续不显示接管入口，避免对不适用的会话调用 `attach`。

## 0.7.0

- 插件变更完成后明确提示当前运行时仍加载旧状态，可一键重启当前项目运行时并重新加载插件与市场列表。

## 完整功能

- MCP 配置文件按 CodeBuddy 的 JSONC 路径规则读取，界面只展示环境变量和请求头的键名，不返回其中的凭据值。
- MCP 写操作绑定发起时的项目和运行时代次；运行时未连接时仍可查看配置，但禁用添加、删除和工具读取。
- GUI 更新下载仅接受本仓库正式 Release 下版本号与文件名一致的 `CodeBuddy-GUI-Setup-<version>.exe`，安装仍由用户主动确认。
- Windows 窗口最小尺寸从 1200×760 调整为 900×640，支持小屏、远程桌面和左右分屏。
- 展开侧边栏会在 220px 到 252px 之间响应式收缩，折叠宽度保持 60px。
- 工作区文件树和 Git 变更列表改为带下限的响应式宽度，为编辑器和 diff 保留更多空间。
- 工作区侧栏在 280px 到 360px 之间随窗口变化，变更列表在 300px 到 380px 之间变化；现有横向滚动和主区域 `min-w-0` 行为继续保留。
- 默认窗口仍为 1440×920，已有窗口位置和最大化状态继续恢复；仅放宽用户主动缩小窗口时的下限。
- 每个终端面板独立维护帮助弹层状态；弹层打开时保留原面板焦点行为，不会误切换或操作终端会话。
- 原有悬停查看继续保留，键盘聚焦时也会显示帮助内容；图标对辅助技术隐藏，按钮提供明确名称。
- 渲染入口统一序列化 Error、字符串和对象形式的拒绝原因，并监听捕获阶段的资源错误及全局 `unhandledrejection`。
- 同类型、同消息和同堆栈的错误在 2 秒内只上报一次，去重记录会自动清理；主进程继续限制字段长度并在落盘前脱敏。
- 每个会话拥有独立的 350ms 草稿保存计时器；切换输入会刷新本会话计时器，不会影响其他会话的草稿状态。
- 会话发送、状态更新等即时持久化路径会吸收待处理草稿保存，避免随后重复写入；最终退出保存继续覆盖全部当前草稿。
- 托盘“完全退出”现在先处理未保存编辑内容，再保存当前工作区并执行最终同步快照；只有快照确认成功才通知主进程退出。
- 退出保存链发生异常、同步 IPC 不可用或主进程拒绝写入时，应用保持打开并显示失败原因，不再静默丢失最后状态。
- 渲染错误边界会向主进程报告错误消息、堆栈、组件堆栈和当前路由；日志写入前会脱敏并在超过 1MB 时保留尾部，避免长期增长。
- 渲染进程异常退出会记录 Electron 提供的原因和退出码，并由主进程提供恢复对话框；React 崩溃页可直接打开用户数据目录获取 `crash.log`，打开失败时保留明确反馈。
- 聊天输入区可直接拖放本地文本文件和图片；读取期间会锁定重复操作，失败原因会显示在原会话，拖放结果不会因项目或会话切换写入错误页面。
- 设置页复制用户数据目录等系统路径时使用统一剪贴板兼容回退，避免安装版或受限环境下 Clipboard API 不可用导致无响应。
- 设置页可一键导出 JSON 诊断报告，包含应用、系统、窗口、运行时状态和最近启动/崩溃日志；密码、Token、API Key、用户目录和项目路径会脱敏，不读取对话、草稿、项目文件或 `product-state.json`。
- 设置页提供 CodeBuddy GUI 手动更新检查：显示当前与最新版本、区分尚无正式发布与网络失败，15 秒超时后可重试；可直接打开本仓库对应版本的 Windows 安装包下载地址，不会后台静默下载或自动安装。
- 窗口隐藏或未聚焦时，CodeBuddy 后台任务完成或失败会发送桌面通知；待发送队列只在最终结果时通知，用户取消不误报失败，点击通知会恢复窗口并打开对应对话，设置页可随时关闭此功能。
- CodeBuddy GUI 的主题、提示建议、剪贴板贴图和 Token 计数现在独立保存在本机，不再写入 CodeBuddy CLI 项目设置；旧版本共用缓存会自动迁移，切换项目不会重置这些偏好。

- 首次关闭窗口到托盘时会明确提示应用仍在后台运行，提示状态随窗口配置持久化；托盘菜单使用“打开 CodeBuddy GUI”和“完全退出”区分隐藏与真正退出。
- 聊天消息和 Worker 日志复制支持 Clipboard API 与兼容回退，成功和失败都会在原按钮上反馈，并在页面卸载时清理状态计时器。
- 设置页可通过主进程直接打开安装版用户数据目录，便于获取诊断日志和本地状态；应用信息加载、路径复制和目录打开均提供明确成功或失败反馈。
- 当前项目要求登录时，登录页会显示项目身份，并允许切换已添加项目、打开其他项目或重启当前运行时；项目与运行时先完成切换，再由新项目重新执行认证和会话初始化。
- 服务状态确认失败时可直接重启当前项目运行时并重试，或打开其他项目继续工作；工作区选择会区分成功、失败和用户取消，不再把恢复页变成只能重复重试的死路。
- 快捷键页明确区分 CodeBuddy GUI 与 CodeBuddy CLI：GUI 快捷键保存在本机并即时驱动侧边栏、新建对话及主要页面导航，支持按键捕获、重复与系统保留组合键校验和独立恢复默认；CLI 快捷键继续绑定当前项目运行时并保留原有组合序列语义。
- 多项目和多对话持久化，支持并行 CodeBuddy 运行时与后台会话。
- 文件浏览、Monaco 编辑器、项目终端和 Git 工作流。
- 插件、Worker、任务、实例、日志、指标、链路和远程渠道管理。
- Windows 托盘、单实例唤回、窗口状态恢复和可选安装目录。
- 次级页面按需加载，编辑器仅在进入工作区后加载。
- 首次启动时若 CodeBuddy CLI 不可用或项目目录已移动，会显示可操作的明确错误信息。
- 应用界面、Windows 产物和 ACP 客户端上报统一使用同一份产品名称与版本。
- 使用统计适配当前 CodeBuddy 数据契约，可查看会话、消息、活跃天数、模型 Token、常用工具和近期活动。
- 远程渠道仅显示当前 CLI 支持的操作；微信解绑后可直接重新扫码绑定。
- 系统监控展示当前 CLI 的实例健康、PID、目录、版本、运行时长和内存占用。
- 插件页兼容单个插件响应，并展示安装范围、作者、许可、技能数量和市场信息。
- Worker 页展示当前实例、Endpoint、模式和心跳，并避免误终止当前项目运行时。
- 快捷键管理支持搜索、上下文筛选、用户覆盖、后端校验、恢复单项默认和全部重置。
- 设置页补齐 Token 计数、通知、信任、状态栏和可信目录，并正确保存与回滚嵌套设置。
- 设置页明确区分 CodeBuddy GUI 与 CodeBuddy CLI 配置，移除误导性的 GUI 语言选项，并在项目设置重载期间保持主题稳定。
- 项目重命名和移除改为应用内确认流程，明确提示未保存修改，并在状态持久化失败时恢复操作前状态。
- Git 变更页使用应用内分支选择、新建分支和丢弃确认流程，并在操作失败时保留上下文与错误信息。
- 插件卸载和市场删除使用应用内确认与失败反馈，并阻止多个插件写操作同时执行导致状态互相覆盖。
- 远程渠道解绑和删除使用应用内确认与失败保留，删除微信实例时会同步停止对应二维码轮询。
- 定时任务删除、Worker 终止和快捷键全量重置使用统一应用内确认，并在失败时保留操作上下文。
- 文件重命名和删除改为应用内确认，目录操作会同步映射或关闭其中已打开文件，并明确保护未保存内容。
- 切换项目、工作区、目录或文件时，未保存编辑内容统一使用应用内确认，不再弹出系统原生确认框。
- 从系统托盘真正退出应用时会检查当前编辑器的未保存内容，取消确认即可继续留在应用中。
- 终端页会明确展示项目运行时停止或启动失败状态，并支持原地重新启动；不可执行的终端关闭操作会正确禁用。
- 对话页在运行时或会话连接失败时直接展示错误与恢复入口，未连接期间保留草稿但阻止无效发送。
- 消息发送失败时会恢复原草稿和附件，队列消息也由统一发送事务负责清空与失败回滚。
- 会话删除使用统一应用内确认，删除失败时保留弹窗与错误以便重试，并仅在后端删除成功后释放本地连接。
- Git 变更页按工作区和文件隔离异步结果，快速切换项目或文件不会被旧请求覆盖；提交和暂存按钮会匹配真实索引状态。
- Git 变更页可预览未跟踪文本文件，并对空文件、二进制内容和超长文件提供明确边界提示。
- 编辑器保存请求按项目、文件和请求序号隔离，保存期间切换文件不会再污染新文件的保存基线，保存结果会在编辑器内明确反馈。
- 工作区会监听磁盘文件变化并自动刷新未修改文件；存在本地编辑时会保留当前内容，并提供重新载入或继续编辑的明确选择。
- Electron 请求代理会保留图片二进制响应并提供标准 Headers、Blob 和 ArrayBuffer 读取能力，微信二维码可在安装版中正确显示。
- 远程渠道的加载、创建和二维码轮询按项目隔离；切换项目会清空企微凭据输入，旧项目请求不会再覆盖当前页面。
- 同一项目的运行时启动、停止和重启会串行执行，认证请求固定绑定对应运行时地址，项目切换期间不会串用密码或 Token。
- 项目运行时页支持不同项目并行操作、独立成功与失败反馈、后台状态对齐和持续更新的运行时长，旧刷新结果不会覆盖新状态。
- 从项目运行时页启动或重启当前项目时会重新连接当前会话并刷新运行时数据；停止后相关会话显示为已断开，持久化的待发送队列在重连时会保留并继续处理。
- 设置加载会区分系统信息与设置请求的真实失败，并提供明确重试反馈；切换项目会清空旧项目的设置视图。
- 设置写入固定绑定发起操作时的项目地址与认证上下文，模型和模式切换也只更新原会话，避免快速切换项目或会话时互相覆盖。
- 日志页切换项目时会重新加载 Worker，并按项目、Worker 和请求序号隔离日志结果；自动刷新产生的迟到响应不会覆盖当前选择。
- 插件安装、卸载、启停和市场增删只会收尾发起操作的项目；插件与市场错误分开呈现，不再互相覆盖。
- 定时任务创建、删除和刷新绑定原会话，切换会话会清理旧操作反馈，迟到结果不会修改新会话表单或错误状态。
- Worker 与 Daemon 的刷新、启停、重启和终止反馈按项目隔离，旧项目操作完成后不会刷新或关闭新项目的页面状态。
- 快捷键校验、保存和全量重置绑定发起操作的项目；校验期间切换项目会停止后续写入，旧请求不会修改新项目页面。
- Git 暂存、丢弃、分支、拉取、推送和提交操作固定绑定点击时的仓库目录，多步丢弃过程中切换项目也不会把后续命令执行到其他仓库。
- 切换分支、Pull 和丢弃修改会先保护编辑器未保存内容，成功后重新载入或关闭受影响文件；新分支首次 Push 会自动建立 origin upstream，Diff 视图也会跟随 GUI 主题。
- PTY 创建和释放固定绑定原项目运行时与认证上下文，切换项目后的迟到会话会被清理，旧终端输出、状态和重连结果不会写入新项目。
- 会话重命名和删除固定绑定发起操作的项目，旧项目操作完成后不会关闭或覆盖新项目的侧边栏状态。
- 侧边栏模型与模式切换按项目和会话隔离并显示结果；内联重命名会阻止回车与失焦造成的重复提交。
- 新建会话会统一锁定侧边栏和标题栏入口，避免快速点击创建多个空会话；状态保存失败会回滚，切换项目期间不会初始化错误的会话。
- 模型和模式写入按会话串行执行，聊天页会显示切换进度与真实结果，失败时保留选择菜单供用户重试。
- 权限响应、问题回答和停止生成固定绑定原会话并阻止重复请求；处理后的卡片会持久显示结果，失败时保留内容供重试，问题卡支持后端选项。
- 附件选择结果绑定原会话并自动去重；文件改为逐项异步读取，单个无效或超限文件不再拖垮整批选择，并补齐常见源码与配置格式。
- 项目、会话和工作区切换采用最新导航生效机制，旧流程不会在运行时启动或目录加载后初始化错误会话；侧边栏显示切换目标与失败信息。
- 项目或会话切换完成前会同步锁定侧边栏和标题栏的新建会话入口，避免在半初始化项目上插入空会话。
- 项目重命名和移除改为独占状态事务；删除活动项目时固定替代项目与会话，失败会恢复原项目连接，完成后再开放导航入口。
- 登录、认证刷新和启动流程按请求代次与项目地址隔离；退出登录会使在途认证及会话初始化失效，迟到响应不再恢复旧凭据。
- 产品状态保存改为串行写入并在执行时抓取最新快照；旧保存回包不再覆盖正在编辑的草稿、会话、项目或终端状态。
- 待发送队列随线程持久化，重启后可继续发送；附件仅保存路径并在发送前重新读取、校验，文件失效或能力不匹配时保留队列并明确报错。
- 文件创建、目录创建、重命名和删除绑定发起项目与目录；切换工作区后旧操作不再刷新、清空或报错到新工作区，创建入口会阻止重复提交。
- 文件名搜索结果会按真实类型打开文件或进入目录；新建和重命名前检查当前目录同名项，避免通过写入或移动接口覆盖已有文件，并展示后端返回的具体失败原因；Monaco 编辑器会随 GUI 亮色、暗色或系统主题切换。
- 终端关闭和重连会等待 PTY 释放的真实 HTTP 结果；释放失败时保留 pane 并显示错误，按钮与快捷键共用操作锁，不再静默遗留不可见会话。
- 远程渠道连接、解绑和删除绑定原项目与页面代次，渠道卡片、插件写操作、任务操作和设置编辑器使用同步事务锁，快速重复点击或切换项目后不会重复请求或回写旧页面。
- 快捷键校验、保存和恢复默认使用单一写事务，避免快速重复操作以旧配置覆盖新配置；统计、指标和监控刷新按项目使用同步请求锁，重叠刷新不会提前结束忙碌状态。
- 同一会话的重命名与删除按会话串行执行；侧边栏项目与会话操作、聊天重新连接、停止生成和待发送队列使用同步事务令牌，双击发送不会把同一草稿重复加入队列，切换项目后旧结果不会污染当前聊天。
- 设置页补齐网关认证编辑入口并使用掩码输入，认证值不会写入 GUI 的本地设置缓存；聊天用量显示真正服从“显示 Token 计数”设置。
- “允许剪贴板贴图”现在会在聊天输入框接收 PNG、JPEG、GIF 和 WebP 图片，复用运行时能力与 20MB 限制校验，并保存为可在待发送队列中跨重启恢复的临时附件；90 天前的剪贴板临时图片会自动清理。
- CodeBuddy 提示建议会按当前会话显示在输入框上方，可一键填入草稿或关闭；发送新消息后会清除旧建议，“提示建议”开关现在真正控制界面行为。
- 会话更新会从 CodeBuddy 的 _meta 契约提取提示建议和团队状态；聊天页可查看当前团队成员，并显示此前会被静默丢弃的执行错误、产物、检查点、任务和目标动态。
- 权限在其他客户端处理后会同步关闭对应卡片并保留真实决定；目标进度、目标状态、历史恢复和运行阶段从 CodeBuddy 元数据进入当前会话，避免界面停留在过期的待处理状态。
- 执行 /clear 或 /reset 后，当前 GUI 对话会接管 CodeBuddy 返回的新 sessionId，清理旧上下文和待发送数据，并重新加载新会话配置；加载失败时保留新 ID 和可恢复的明确错误，不再继续向失效会话发送。
- CodeBuddy 的 _codebuddy.ai/question 和 session/request_permission 服务端请求会保留 JSON-RPC 请求 ID 并正确回包；问题卡支持单选、多选、文本答案与取消，标准权限请求和跨客户端权限广播会去重并兼容扩展接口收口。
- 登录状态检查失败不再被误判为已登录，服务不可用时会停留在明确的恢复界面并允许重试，运行中认证失效会返回登录页而不是卡在错误状态；侧边栏、Worker、任务、设置和插件页改为按需订阅状态，流式对话不再触发无关重页面整体重渲染。
- 主进程报告 ACP SSE 超时截断时，渲染层不再把不完整响应视为成功；对话会进入明确错误状态并恢复草稿与附件，方便用户重试。
- ACP 连接更换或会话客户端释放时，旧连接上的 JSON-RPC 权限和问题卡会立即标记失效并取消操作按钮，不再让用户对无法回包的请求继续操作。
- 应用重启、项目运行时更换或异常退出后，终端保留布局和最近输出，但会自动丢弃已失效的 PTY 会话 ID，并在新运行时就绪后重建真实终端。
- 终端尺寸变化按渲染帧合并 PTY resize 请求，日志自动刷新也不再在慢请求期间重复发起后端调用。
- 每个项目会保存上次打开的目录、文件和未保存编辑内容；切换项目或重启应用后会恢复工作现场，明确选择放弃修改时则保存为干净状态。
- 待发送提示队列支持上移、下移和删除，顺序调整与队列内容一起持久化，失败时回滚到操作前状态。

## 使用要求

- Windows x64。
- 本机可直接运行 `codebuddy` CLI，并已完成其必要配置。

## 数据保留

- 覆盖安装会保留现有项目和对话数据。卸载程序默认保留用户数据，方便后续重装恢复。
- 产品状态保留最近一次有效备份；主状态文件损坏时自动恢复备份，并保留原始坏文件用于诊断。
